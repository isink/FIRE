## 1. 快速开始

### 启动应用

项目有**两个版本**，二选一启动即可：

**版本 A：Next.js 14 + shadcn/ui（推荐，新版）**
```bash
cd next-app
npm install   # 第一次
npx next dev  # 启动 dev server
```
浏览器打开 → `http://localhost:3000/`

**版本 B：Vanilla HTML + Flask（旧版，含行情代理）**
```bash
cd data-check
python3 backend.py
```
浏览器打开 → `http://localhost:8000/`

> ⚠️ Vanilla 版需要 Flask（`pip install flask`）+ 装好 `fire_data_check.py` 的依赖。如果只想看建模能力，直接用 Next.js 版即可。

### 三分钟入门

1. **打开 `/app`**（首页点"立即试用"或直接访问）
2. **看默认方案**：左侧 sidebar 切换到「总览」，应用已经预填了一个"30 岁上海工程师"的默认方案
3. **看顶部 6 个 metric 卡片**：净资产 / 已达进度 / FIRE 预计达成 / Coast FIRE / 达成概率 / 储蓄率
4. **调整你自己的参数**：
   - 切到「家庭」改出生年和退休年
   - 切到「收入」改月薪（注意区分"税前 / 税后"）
   - 切到「资产」改持有金额
5. **看 FIRE 时间变化**：每次改动 140ms 后自动重跑 5,000 次蒙特卡洛

### 数据存哪

- **本地优先**：所有输入即时存到浏览器 `localStorage`（key 名 `fire-state-v15`）
- **不需要登录**：演示模式直接可用
- **导出备份**：Topbar 三点菜单 → "导出 JSON"，保存方案文件
- **清空重来**：浏览器 DevTools → Application → Local Storage → 删除 `fire-state-v15`

## 2. 核心概念

### FIRE（Financial Independence, Retire Early）

**财富自由 + 提前退休**。本应用把它具体化为：你的净资产 P50 值首次触及"目标金额"的那一年。

```
FIRE 时间 = 投资组合 P50 中位数路径 ≥ plan.target 的最早年份
```

注意：**FIRE 时间 ≠ 实际退休时间**。你可能 35 岁就 FIRE（账面够了），但选择 45 岁退休（多攒缓冲）。两者由 `家庭`视图里的 `retireYear` 独立控制。

### Coast FIRE（可停投点）

到了 Coast 节点后，**即使停止定投，仅靠现有本金的复利**到目标退休年也能达到 FIRE。

```
Coast 目标 (在时刻 t) = target / (1 + 实际收益率)^剩余年限
```

Coast 之后还在工作 = 完全靠主动收入覆盖生活，资产复利接管增长。

### P10 / P50 / P90（蒙特卡洛分位数）

5,000 次随机模拟得到的 5,000 条净资产路径。每个采样月排序：
- **P10**：10% 最差的运气（10% 的世界你低于这个值）
- **P50**：中位数（一半世界比这好，一半比这差）
- **P90**：10% 最好的运气

FIRE 时间用 P50。成功率用"5,000 条里多少条触及 target"。

### 蒙特卡洛模拟（Monte Carlo）

每个资产独立的随机过程：每月 return = `μ_month + σ_month × Z`（Z 为标准正态随机变量）。

```
单月期望: μ_year / 12
单月波动: σ_year / √12
```

A 股口径预设：
- 现金/余额宝：2.0% / 0.5%
- 公募基金：8.0% / 16%
- A 股 ETF：9.0% / 18%
- 港股：7.0% / 20%
- 美股：10.0% / 16%
- 黄金：6.0% / 15%
- 加密币：30.0% / 80%
- 个人养老金 (IPA)：5.0% / 10%
- 房产：3.0% / 6%

### 四桶取款（Withdrawal Buckets）

退休后取款不是"从总池子里取"，而是按顺序：

```
1. 现金桶（余额宝 / 银行理财）— 优先取，无成本
2. 应税桶（基金 / A股 / 港美股 / 黄金 / 加密）— 次取，按现值比例分摊
3. IPA 桶（个人养老金）— 60 岁后才解锁
4. 房产桶 — 不能日常取（除非加"卖房"事件）
```

### 五险一金 + 个税精算

每条 `税前` 类型的收入流自动算：
- **五险一金**：按城市标准上下限 × 个人缴费比例
  - 上海：基数 7310-36549，养老 8% + 医疗 2% + 失业 0.5% + 公积金 7%
  - 北京 / 深圳 / 杭州 / 广州 各异
- **综合所得 7 级累进个税**（3% / 10% / 20% / 25% / 30% / 35% / 45%）
- **基本减除**：5,000 元 / 月
- **7 项专项附加扣除**：房贷利息 / 租房 / 子女教育 / 婴幼儿照护 / 赡养老人 / 继续教育 / 大病医疗

输出 `实发月收入`，作为现金流注入资产。

### 生命阶段（Life Stages）

把人生切成三段：
- **在职** — 默认 working，月支出按 `expenseCategories` 累加（带分类通胀）
- **过渡期**（可选）— 育儿期 / Gap Year / 半退休。独立月支出 + 收入倍数（0=无 / 0.5=半职 / 1=全职）
- **退休** — 收入停止，按 SWR 或固定月支出取款

家庭层退休时刻 = 所有人 `retireYear` 的最大值。

### 社保养老金（精算公式）

```
基础养老金 = 退休时社平 × (1 + 缴费指数) ÷ 2 × 缴费年限 × 1%
个人账户养老金 = 退休时余额 / 计发月数（60 岁 = 139）
月发放 = 基础 + 个人账户  [仅在 age ≥ 60 时]
退休后跟随通胀微调
```

每个家庭成员独立计算（共享 `plan.pension` 配置，但用各自 `birthYear` 触发 60 岁）。

## 4. 九个视图详解

### 4.1 总览（Overview）

应用核心页面。从上到下：

#### Hero 指标条（6 张卡片）

| 卡片 | 含义 | 计算 |
|---|---|---|
| 净资产 | 当前所有资产总和 | `Σ amountCny` |
| 已达进度 | 当前净资产 / 目标 | `initial / target × 100%` |
| **FIRE 预计达成** | 红色高亮 — 何年达到 target | P50 路径首次触及 target |
| **Coast FIRE** | 黑色高亮 — 何年可停止定投 | P50 ≥ Coast目标(t) 的最早年 |
| 达成概率 | 5,000 路径里有多少条曾触及 target | `successes / RUNS` |
| 储蓄率 | 当下口径 | `(税后收入 − 月支出) / 税后收入` |

> 💡 **储蓄率**会同时显示"退休可持续 X%"，那是"曾达成 FIRE 后仍未破产到模拟终点"的比例。

#### 主图：净资产模拟

可在 **堆叠 / 线条** 之间切换：

- **堆叠模式**（默认）：按"现金 / 应税 / IPA / 房产"四桶分层显示 P50 路径分解。三层之和 = P50 总值
- **线条模式**：P50 中线 + P10/P90 半透明带（用于看不确定性范围）

#### 逐年预测表

P50 中位数路径的逐年数据：
- 年份
- 年收入（税后口径）
- 年支出（含通胀）
- 年偿债（若有债务）
- 年净储蓄
- 投资组合 P50 终值

FIRE 年份会用 🎯 标记。退休后年份字体淡化。

#### 月度现金流桑基

可视化"税前 → 五险一金 + 个税 + 各支出 + 净储蓄"的流向。每条流的宽度 = 金额比例。

#### 退休期取款来源

退休年份起，每年从哪个桶取了多少（堆叠柱状图）。仅在有提取的年份显示。

#### 敏感性 tornado

点 "运行分析" 跑 14 次扰动 sim，输出哪个杠杆最能改变 FIRE 时间。
- 月支出 ±20%
- 月收入 ±20%
- 资产预期收益率 ±20%
- 通胀率 ±1pp
- 初始净资产 ±20%
- 目标金额 ±20%
- 波动率 ±20%

显示为对称双向条，红色 = 推迟 FIRE，绿色 = 提前 FIRE。

> ⏱ 运行时间约 2-5 秒，期间 UI 会冻。

### 4.2 家庭（Household）

管理家庭成员（V1 支持最多 2 人）。

每张人员卡片：
- **姓名**（可改）
- **出生年份**（驱动 IPA 60 岁解锁 + 社保发放）
- **计划退休年**（决定该人的收入何时停止）
- **收入流摘要**（只显示，编辑在「收入」标签）

> 💡 **添加配偶**会自动创建一条 "配偶工资 20K 税前" 流。删除配偶会连同其所有收入流一起删。

**家庭层退休** = max(每人 retireYear)。比如本人 50 岁退休、配偶 55 岁退休，家庭层在 55 岁切换为 retired 模式。

### 4.3 收入（Income）

按人分组的收入流管理。

#### 税务设置卡片（顶部）

可折叠。配置：
- **缴费城市**：上海 / 北京 / 深圳 / 杭州 / 广州 / 自定义
- **专项附加扣除**（6 项）：每项有"快捷填充"芯片
  - 房贷利息：首套 1000
  - 租房：一线 1500 / 二线 1100 / 三线 800
  - 子女教育：每个 2000
  - 婴幼儿照护：每个 2000
  - 赡养老人：独生 3000 / 非独 1500
  - 继续教育：学历 400 / 职业 300

设置变化 → 所有 `税前` 类型收入流的实发金额实时更新。

#### 收入流卡片（按人分组）

每条流：
- **名称**
- **类型 toggle**：税前 / 税后
  - 税前：自动算 五险一金 + 个税，展开"税前 → 五险一金 → 个税 → 实发"明细 + 边际税率 + 实际税负
  - 税后：用户直接输入到手金额
- **月薪 ¥**（税前或税后）
- **年增长 %**
- **开始年 / 结束年**

> 💡 **税前 30K 上海**典型计算：30,000 − 5,250（五险一金）− 2,540（个税）= **22,210 实发**，边际税率 20%。

### 4.4 支出（Expenses）

7 类预设支出 × 每类独立通胀率。

预设（按一线城市比例）：
| 类别 | 占比 | 通胀率 (在 CPI 基础上 +X) |
|---|---|---|
| 住房 | 40% | +0% |
| 餐饮 | 22% | +0% |
| 交通 | 8% | +0% |
| 医疗 | 6% | **+3%**（医疗通胀远超 CPI）|
| 教育 | 7% | **+2%** |
| 娱乐 | 10% | +0% |
| 其他 | 7% | +0% |

> ⚠️ 关键设计：**医疗 / 教育**用独立的高通胀率，这是单一 CPI 模型严重低估的部分。退休后医疗占比上升时这个差异更突出。

可以**自由编辑**：改类别名、调金额、改通胀率、删类别、加新类别。

顶部显示"月支出合计" + "加权通胀率"。

### 4.5 资产（Assets）

每行一个资产 + 三段配置：
1. **类型 select + 名称 + 删除**
   - 类型：现金 / 基金 / 股票 / 黄金 / 加密币 / 港股 / 美股 / 养老金 / 房产
   - 切类型时自动套用该类的默认 ret/vol
2. **代码 + 持仓金额 + 月定投**（非房产类）/ **模式 + 估值 + 物业费 + 租金**（房产）
3. **预期年化 + 年化波动率 + 桶标签**

#### 房产特殊配置

- **自住 vs 出租** 切换
- **当前估值** + **月物业费**
- 出租模式额外显示"月租金"
- Sim 中：
  - 自住：每月 `−物业费`
  - 出租：每月 `+租金 − 物业费`
  - 资产本身按 3% 升值率自然增长
  - **房产桶 ≠ 应税桶**：日常退休取款不可动，只能通过"卖房"事件转为现金

#### 桶标签（视觉关键）

| 标签 | 含义 |
|---|---|
| 现金桶 · 优先取款 | 取款顺序最优先 |
| 应税桶 | 次取，比例分摊 |
| IPA 桶 · 60 岁后取 | 锁定，age ≥ 60 才解锁 |
| 房产桶 · 不可日常取 | 永久锁定（除非卖房事件）|

---

### 4.6 债务（Debts）

房贷 / 车贷 / 信用贷等。

每条债务：
- **名称**
- **本金 ¥**
- **年利率 %**
- **年限**
- **还款方式**：等额本息 / 等额本金
- **开始年**

应用自动算月供 + 余额。Sim 中每月从现金流扣除月供直到还清。还清后该笔月供自动转为可投资金。

> 💡 **公积金贷 + 商业贷**通常分两笔加，因为利率不同（公积金 ~3.1% / 商贷 ~3.5-4.5%）。

#### 等额本息 vs 等额本金

- **等额本息**：每月还款金额固定，前期利息占比高
- **等额本金**：每月还本金固定，前期总还款多但总利息少

---

### 4.7 阶段（Stages）

三段时间线 + 社保医疗卡。

#### 时间线条

蓝色（在职）→ 黄色（过渡期，可选）→ 绿色（退休）。
按时间长度比例自动伸缩。

#### 三张阶段卡

- **在职** — 默认月支出（留空 = 用全局月支出参数）
- **过渡期**（默认禁用）
  - 启用时自动填默认 `startYear = 今年+20` 和 `incomeMultiplier = 0.5`
  - 月支出：可设育儿期 +30%-50% 等场景
  - 收入倍数：0 = 无 / 0.5 = 半职 / 1 = 全职
- **退休** — 月支出（留空 = 用全局退休后月支出）
  - **家庭退休年是只读**：派生自「家庭」视图各人 `retireYear` 的最大值

#### 社保养老金 + 医疗缺口卡

可启用。配置：
- **已缴年限**（截至今年）
- **缴费指数**（1.0 = 按社平缴费）
- **当前社平 ¥/月**（所在城市，默认上海 11000）
- **个人账户余额 ¥**
- **退休医疗缺口 ¥/月**（按医疗通胀 CPI+3pp 复合）

启用后右上角显示"60 岁起 ¥X,XXX/月"实时预估。每个家庭成员按各自 birthYear 触发 60 岁。

### 4.8 目标（Goals）

中间大额支出独立管理。每个目标显示其对 FIRE 时间的影响。

#### 添加目标

四个预设按钮：
- 🏠 买房首付（默认 150 万 @ 5 年后）
- 🎓 子女教育金（默认 50 万 @ 18 年后）
- 🚗 换车（默认 30 万 @ 4 年后）
- ✈️ 海外留学（默认 80 万 @ 20 年后）

或点 "+ 自定义" 开 modal 任意填。

#### 每个目标卡片

| 字段 | 含义 |
|---|---|
| 优先级 chip | 必须 / 希望 / 可选（颜色区分）|
| 目标金额 | 一次性支出额 |
| 目标年份 | 距今 X 年 |
| 届时 P50 | 无目标基线下，该年的净资产 P50 值 |
| 承担能力 badge | 覆盖率 ≥ 200%: ✓ 充裕; ≥ 100%: ✓ 可覆盖; ≥ 60%: ⚠ 紧张; < 60%: ✗ 不足 |

#### 启用/暂停 toggle

每个目标可独立"暂停"（保留卡片但不参与 sim），用于做"假如不要这个目标 FIRE 会快多少"的对照。

#### 顶部聚合指标

- 规划数（启用中的目标数）
- 总规划支出
- **FIRE 推迟**（与"全部暂停"的基线 sim 对比）

> 💡 **典型数字**：上海 30K 月薪 + 80W 初始的人，加 "买房首付 150 万 @ 5 年后" 目标，FIRE 时间从 16 年 推迟到 22 年（+4.25 年）。

---

### 4.9 事件（Events）

时间线上的一次性 / 持续性现金流调整。

#### 与「目标」的区别

- **目标**：有优先级 + 承担能力 badge + 独立 FIRE 影响分析，**单向支出**（amount 总是正数）
- **事件**：更通用，可正可负，可一次性可持续。常用于：
  - 持续性社保领取（每月 +X）
  - 育儿期持续多支出（每月 −X）
  - 大额遗产 / 中奖（一次性 +X）

#### 三种预设按钮

- 🏠 买房首付（一次性 −150 万 @ 5 年后）
- 👶 生娃加支出（持续 −5,000 /月 @ 3 年后）
- 🏦 社保领取（持续 +3,000 /月 @ 30 年后）

#### 事件 modal 字段

- 名称
- 触发年份
- 类型：一次性 / 持续性
- 金额 ¥（正 = 收入，负 = 支出）

## 6. 三种退休提取策略

通过"安全提取率 (SWR)"上方的下拉切换（旧版在右侧参数面板）：

### 1. 固定金额（通胀调整）

```
月取款 = 月支出 × (1 + 通胀)^t
```

不管市场涨跌，固定按通胀调整。**风险**：碰到大跌就会持续高比例取款，可能耗尽。

### 2. 固定比例 SWR%

```
月取款 = 当前组合 × SWR / 12
```

按当前组合的固定百分比取。**风险**：组合跌时取款少，可能不够生活。

### 3. Guyton-Klinger（动态护栏）

经典论文级动态策略：
- 起始：`年取款 = 退休时组合 × SWR`
- 每年三条规则：
  - **资本保全**：当前提取率 > 初始 × 1.2 → 砍 10%
  - **繁荣规则**：当前提取率 < 初始 × 0.8 → 涨 10%
  - **通胀规则**：上一年组合负回报 → 跳过本年通胀调整

> 💡 GK 在 A 股高波动场景下持续率比固定策略高 5-15%。

## 8. 五个真实场景教程

### 场景 A：25 岁互联网工程师 · 单身 · 上海

**配置**
- 家庭：1 人，1999 出生（现 25 岁），50 岁退休
- 收入：35K 税前月薪
- 资产：现金 12 万 + A 股 ETF 0 + 公募 0
- 目标：500 万

**调整步骤**
1. 「家庭」: 改 birthYear 到 1999, retireYear 到 2049
2. 「收入」 (本人): 改月薪 35,000 (税前)
3. 「资产」: 删除多余资产, 留一个 cash 改余额 120,000, 加一个 stock (沪深300ETF) 余额 0、月定投 5000
4. 「参数面板」: target 滑块拉到 500 万

**预期结果**
- FIRE 时间：~15 年（40 岁）
- 成功率：>95%
- 关键约束：IPA 完全没用（要 60 岁解锁，FIRE 在 40 岁），所以前 20 年退休全靠应税桶

**优化建议**
- 提高储蓄率 → 「敏感性 tornado」会显示"月收入"和"月支出"是最强杠杆
- 加入「目标」 → 教育金 / 买房（如有计划）测试 FIRE 推迟多少

### 场景 B：32 岁夫妻 · 深圳 · 计划生娃

**配置**
- 家庭：2 人。本人 1993 年生（产品经理 30K），配偶 1994 年生（设计师 22K）
- 计划：3 年后生 1 孩，育儿期支出 +5,000/月 持续 18 年
- 买房：4 年后 200 万首付
- 目标：1000 万

**调整步骤**
1. 「家庭」: 默认本人，添加配偶。设两人 birthYear / retireYear
2. 「收入」: 本人改 30K 税前。配偶改 22K 税前
3. 「税务设置」: 上海或深圳，子女教育扣除暂为 0（孩子还没生）
4. 「目标」: 添加 "买房首付" 预设，金额 200 万，年份改为 +4 年
5. 「事件」: 添加 "生娃加支出"，年份 +3 年，月度变化 −5000
6. 「参数面板」: target → 1000 万

**预期结果**
- 无目标基线 FIRE：~12 年
- 加买房首付：推迟 ~3-4 年
- 加育儿期支出 18 年：再推迟 1-2 年
- 最终 FIRE：~15-18 年（47-50 岁）

**敏感性分析**：通常"配偶薪资"和"目标金额"是最强杠杆。

### 场景 C：35 岁高薪程序员 · 北京 · 房贷在身

**配置**
- 家庭：1 人，1990 年生
- 收入：60K 税前
- 资产：现金 30 万 + 沪深300ETF 80 万 + 自住房（市值 600 万）
- 债务：房贷 400 万 / 利率 3.5% / 30 年 / 等额本息（剩 25 年）
- 目标：1500 万

**调整步骤**
1. 「家庭」: birthYear 1990
2. 「收入」: 60K 税前。「税务设置」: 北京，房贷利息 1000
3. 「资产」: 加 "房产" 类型，自住模式，估值 600 万，物业费 800/月
4. 「债务」: 添加房贷，本金 400 万，利率 3.5%，年限 25
5. 「参数面板」: target → 1500 万

**预期结果**
- 房贷月供约 20,000，每月现金流被吃掉 1/3
- FIRE 时间被房贷拖累约 3-5 年
- "退休前还清房贷" 的对比：在「债务」改年限到 15 看 sim 怎么变

**关键洞察**：高利率 + 高本金的房贷对 FIRE 影响巨大。3.5% 利率下其实不糟，但如果是 5%+ 的早年商贷 + 大本金，结论会逆转。

## 9. FAQ

### 数据安全 / 隐私

**Q：我的财务数据存在哪？**
A：本地优先 — 所有数据存浏览器 `localStorage`。未登录时**不会上传任何地方**。Next.js 版本暂未接云端同步（vanilla 版有 Supabase 接口，但需要自己配 supabase-config.js 才生效）。

**Q：我清浏览器缓存会丢数据吗？**
A：会。清 localStorage = 全部清空。**定期 Topbar → 导出 JSON 备份**。

**Q：我能在多设备同步吗？**
A：目前只能手动 导出/导入 JSON。Cloud sync 在 vanilla 版有 Supabase 实现框架，但需要你自己注册 Supabase 项目并填配置文件。

### 建模质量

**Q：预期收益率怎么定的？合理吗？**
A：A 股 2005-2024 含分红口径 + 行业惯例。每个资产可单独调整。蒙特卡洛输出 P10/P50/P90 三档，不是单一确定性数字，已经计入不确定性。

**Q：通胀率应该填多少？**
A：默认 2.5%（中国近 10 年 CPI 平均）。**关键：医疗和教育子类别已自动 +3% / +2% 高通胀**。如果你预期未来 30 年通胀回升，全局通胀拉到 3-3.5%。

**Q：5,000 次蒙特卡洛够吗？**
A：对于 P10/P50/P90 三档来说足够，置信区间已经稳定。10,000 次差距很小但耗时翻倍。

**Q：模拟年数填多少？**
A：默认 30 年。建议填到你的预期寿命 − 当前年龄。35 岁预期活到 85 = 50 年。**注意：模拟年数太短会让"退休可持续性"虚高**（还没破产就到模拟终点了）。

### 个税精算

**Q：我有多个收入来源（工资 + 副业），怎么算个税？**
A：当前实现是**每条流独立预扣**（按各条流单独算）。这接近"每个用人单位单独预扣"的实际情境。但年终汇算清缴时综合所得是合并计税，可能与实际有几个点的偏差。

**Q：年终奖怎么处理？**
A：暂未单独支持。把年终奖按月平摊到税前月薪里近似。

**Q：自由职业 / 经营所得呢？**
A：用"税后"类型，自己算好实发金额输入。

### 五险一金

**Q：我用的不是上海五险一金，怎么改？**
A：「收入」→「税务设置」展开 → 缴费城市下拉。支持上海/北京/深圳/杭州/广州。其他城市选"自定义"（暂未真正暴露配置 — V2 任务）。

**Q：基数上下限太低 / 太高怎么办？**
A：默认基数是 2024 标准。如果你想精确建模，自己手动调资产里的 ret/vol 数字来抵消基数差异。

### IPA（个人养老金）

**Q：IPA 真能 60 岁前永远不能取吗？**
A：法规上对身故 / 移民 / 重病等有特殊情形。Sim 把它简化为"60 岁前永远锁住"。如果你打算钻空子 60 岁前取，自己手动减少 IPA 的 amountCny。

**Q：IPA 12,000/年上限怎么建模？**
A：默认 IPA 资产的月定投 dcaAmount 是 1,000（= 12,000/年）。Sim 当前**不强制**这个上限（你可以填更高数字）— 自己确保不超即可。

### Coast FIRE

**Q：Coast FIRE 比 FIRE 早 N 年，是不是说我可以"半退休"了？**
A：Coast = 即使**完全停止主动投资**（不再定投），仅靠现有本金的复利到目标退休年也能达到 FIRE。你仍需要主动收入覆盖**日常开销**（不能动本金），但**可以停止增量储蓄**。

**Q：Coast FIRE 没有数字（显示 — ）？**
A：要么实际收益 < 通胀（实际负收益），要么模拟年数太短组合还没到 Coast 临界。

### 退休提取

**Q：SWR 3.5% 是怎么来的？**
A：经典 Trinity Study 是美股 4%。A 股波动率高于美股 + 长期回报不一定可比，我们用 3.5% 更保守。可在右侧参数面板 SWR 滑块调。

**Q：Guyton-Klinger 适合所有人吗？**
A：适合"愿意根据市场调整生活水平"的人。如果你坚持要"无论市场怎样，退休后每月固定 X 万"，用"固定金额"策略。GK 在 A 股波动场景下更稳健，但要求你接受经济下行时"砍 10% 生活费"。

### Bug / 异常

**Q：「敏感性 tornado」点击运行后页面卡死？**
A：正常 — 同步跑 14 次 5,000 路径模拟，要 2-5 秒。期间 UI 冻结。等就行。

**Q：滑块拖动 sim 不更新？**
A：sim 有 140ms 防抖。拖完松手等一会儿。也可能浏览器 console 有 error，按 F12 看一下。

**Q：图表显示空白？**
A：可能 Chart.js 在容器宽度 0 时初始化。切换 view 再切回来通常解决。

## 10. 局限性与免责

### 已知局限（透明陈述）

#### 建模简化
- **税务**：综合所得用"年化方法"近似（每月按当时收入水平进档），不是真实的年度累计预扣
- **五险一金基数**：固定按 2024 标准，不模拟基数年度调整
- **房产**：升值率固定（不模拟周期），不含房产税 / 维修大修 / 转售税
- **社保**：基础养老金 + 个人账户两部分，未含过渡性养老金、视同缴费
- **医疗**：单一"医疗缺口"参数，未细分门诊 / 住院 / 自费药 / 重大疾病
- **教育金**：用 +2% 通胀单类别处理，未单独追踪每个孩子的支出曲线

#### 数据简化
- **资产相关性**：每个资产独立随机过程，未模拟跨资产相关性（实际危机时多类资产同跌）
- **历史回测**：仅 CSI 300 月度数据 block bootstrap，未含 2008 / 1929 / 1973 等具名情景
- **极端事件**：黑天鹅 / 战争 / 货币危机不在模拟范围

### 关于"FIRE"

财富自由提前退休 (FIRE) 不是适合每个人的目标。它要求长期高储蓄率 + 长期投资纪律 + 一定的运气。**在追求 FIRE 之前请认真思考**：
- 你是否真的想"退休"？还是只是讨厌当前工作？
- 提前 10 年退休 vs 多干 10 年但生活质量更高，你选哪个？
- 你的伴侣 / 家人是否认同这个目标？
- 退休后做什么？你有 30+ 年的余生需要打发

FIRE 是工具，不是目的。本应用帮你算清楚账，但**人生方向问题需要你自己回答**。

隐私政策

最后更新：2026-05-14

注册时：邮箱 + 密码（密码经 Supabase Auth 加盐哈希存储，我们不能看到原始密码）。

使用时：你在应用中输入的财务数据（资产、收入、支出、目标等）。这些数据存储在你浏览器的 localStorage 和你在 Supabase 项目下的 plans 表中。

  • 邮箱：用于账户验证、登录、密码重置、关键服务通知
  • 财务数据：仅用于在你自己的浏览器中渲染图表和模拟；不会被分析、不会被用于训练 AI、不会被聚合统计

Postgres 行级安全（RLS）策略保证每个用户仅能访问 user_id = auth.uid() 的行。即使数据库被读取，单一用户的 access token 也只能拉到自己的方案。

  • 不出售用户数据给任何第三方
  • 不向广告网络共享数据
  • 不未经允许向第三方传输财务信息
  • 不在产品内嵌入第三方追踪脚本（无 GA / 无 Meta Pixel）
  • 在「账号设置」页面导出全部方案（JSON）
  • 删除你的账号（云端方案随之 cascade delete）
  • 通过 support@example.com 申请其他数据请求

本服务仅使用必要的功能性 storage（localStorage 存方案 + sessionStorage 存演示模式标记 + Supabase session cookie）。无分析 cookie、无广告 cookie。

实质性更新会通过登录后弹窗 + 邮件通知。继续使用服务即视为接受新政策。
